# `/company-agent` 重新设计提案

> 仅文档，不改动代码。提交给用户 review 后再决定是否实施。

---

## 1. 现状诊断（"为什么需要重做"）

### 当前页面承担的职责（从代码读出）

`CompanyAgentPage.tsx` (1616 行) 在同一个页面里挤了 **5 个角色**：

| 角色 | 现在长什么样 | 问题 |
|------|--------------|------|
| **Watchlist 管理** | 左 18% 一列 tab + 卡片 | 当我专注分析时它一直占左侧屏幕 |
| **K 线 / Line 图表** | 中 42% TradingView | 信息密度高，但不是"分析的主语" |
| **历史分析列表** | 右 40% 9 列网格 | 是真正的入口，但被挤在角落 |
| **正在跑的分析（streaming）** | overlay 的 ThinkingTimeline + ReportPanel | 流式状态和最终态用同一容器，体验割裂 |
| **AI 推荐 / 任务调度** | 顶部抽屉 + 左 tab | 信息触点散，发现成本高 |

代码里有 3 套布局（Layout A / B / C），全靠 `hasSelection && hasTimeline` 三态切换。**用户每次进来要先想"现在是什么模式"**——这是页面体验最大的问题。

### 用户真正的工作流程

| 步骤 | 用户想做的 | 当前障碍 |
|------|------------|----------|
| 1 | 找一支股票发起分析 | 搜索条浮在底部 65% 位置，半透明，不显眼 |
| 2 | 分析跑的过程中观察思考链 | 流式展示挤在右侧报告区，K 线被冻结时占着大块面积没用 |
| 3 | 拿到结论后想"记下来 / 关注" | 没有"加入关注"动作；watchlist 是单独维度 |
| 4 | 跨多次分析对比同一只股票 | 历史列表是日记式的，不是按 symbol 聚合 |
| 5 | 跨模型看对同一只股票的分歧 | 有 compareMode，但需要先在搜索条切换；不直观 |

## 2. 设计理念

继承 `/dashboard` 的 **Editorial Finance（编辑式财经）** 美学，但 `/company-agent` 不是"日报"，它是**研究室**：

- **Dashboard** = 头版（一眼速读：今天怎么样）
- **Company Agent** = 研究档案（一只股票一份卷宗，可以翻、可以对比、可以追溯）

类比：彭博终端 + FT Lex 专栏 + 学术档案室。

**核心 metaphor**：**"档案柜"（Filing Cabinet）+ "正在写的稿子"（Active Draft）**。
- 档案柜 = 你研究过的所有公司，按 symbol 聚合，每个 symbol 一格抽屉
- 正在写的稿子 = 当前 streaming 中的分析，进入沉浸全屏

## 3. 信息架构（重构后）

页面分两层视图，用 URL 区分：

| URL | 视图 | 内容 |
|-----|------|------|
| `/company-agent` | **档案柜** | 公司列表（按 symbol 聚合）+ 当前没在跑就显示这个 |
| `/company-agent/:symbol` | **公司卷宗** | 单只股票的所有历史分析、K 线、跨模型对比 |
| `/company-agent/:symbol/:runId` | **单次分析报告** | 现在的 ReportPanel，全屏沉浸 |
| `/company-agent/new` | **发起分析** | 全屏向导，输入框居中 + 模型选择 + 历史推荐 |

切换通过浏览器后退/前进 + 顶部面包屑实现。

## 4. 各视图设计

### 视图 A：档案柜（`/company-agent`）

**布局**：编辑式两栏，左 watchlist 浓缩成竖直 tab，主区是 **公司卡片网格**，每张卡片是一只你研究过的股票。

```
┌──────┬──────────────────────────────────────────────────────┐
│ 公司  │  公司档案柜             [+ 发起新分析]               │
│ 指数  │  ─────────────────────────────────────────────────  │
│ 加密  │                                                      │
│ —    │  ╭─ GOOGL ──╮  ╭─ TSLA ──╮  ╭─ NVDA ──╮             │
│      │  │$373.20   │  │$250.40  │  │$210.70  │             │
│ + 推荐│  │ +19.25%  │  │ -3.40%  │  │ +13.4%  │             │
│  3   │  │          │  │          │  │          │             │
│      │  │ 3 份分析 │  │ 5 份分析 │  │ 1 份分析 │             │
│      │  │ 最新 BUY │  │ 最新AVOID│  │ 最新WATCH│             │
│      │  ╰──────────╯  ╰──────────╯  ╰──────────╯             │
│      │                                                      │
│      │  ╭─ TSM ────╮  ╭─ ACWI ──╮  ...                      │
└──────┴──────────────────────────────────────────────────────┘
```

每张 **公司卡片**（300×220）显示：
- 大写 symbol（Fraunces italic 28px）+ 现价 + 当日涨跌
- 该 symbol 的分析次数 + 最新 verdict 徽章 + 最新置信度
- 模型分布小条（每个跑过的模型一格，按颜色编码）
- 最近一次分析的时间（"3 天前"）
- hover 整张卡片亮一档

**点击卡片 → 进入卷宗（视图 B）**

### 视图 B：公司卷宗（`/company-agent/:symbol`）

按"公司专题"组织，单页轮播（继承 dashboard 的左右切换模式），3 张 slide：

#### Slide 1：**当前判断**
- 公司基础信息（名称、行业、当前股价、市值）
- **聚合 verdict**：跨多次分析的"模型共识"——比如 GOOGL 的 5 份分析里 3 BUY / 2 WATCH，怎么呈现这种差异
- K 线图（半屏）+ 关键日期标注（每次发起分析的时间点 + 当时模型 verdict）
- 当前持仓信息（如果 SNB 里有这只股票，显示均价 + 收益率）

#### Slide 2：**分析档案**
- 所有跑过的分析按时间倒排
- 每条一行：日期 / 模型 / verdict / 置信度 / 关键引用数 / 完成耗时
- 点击进入单次报告视图（视图 C）
- 顶部 facets：模型筛选 / verdict 筛选 / 时间筛选

#### Slide 3：**模型分歧**
- 同一只股票，多模型并排对比矩阵
- 行 = 7 个 gate；列 = 模型
- 每格 = 该模型在该 gate 上的核心结论（短句）+ 置信度
- 一眼看到哪些 gate 上模型有共识、哪些有分歧
- 点击一格展开详细对比

### 视图 C：单次报告（`/company-agent/:symbol/:runId`）

**保持当前 ReportPanel 的内容**，只是改 chrome：
- 顶部 breadcrumb：`档案柜 / GOOGL / 2026-04-30 deepseek-v3.2 报告`
- K 线移到右侧 30%（不再是同等地位）
- 报告占左 70%，沉浸阅读
- 引用 chips 和 SourcesPanel 保留
- 底部：进入"模型分歧"对比 / 重新分析 / 分享 入口

### 视图 D：发起分析（`/company-agent/new`）

**全屏空白，输入框居中**，类似 ChatGPT 首页。

```
              ╔═══════════════════════════════╗
              ║                                ║
              ║        发起公司研究             ║
              ║                                ║
              ║   ┌─────────────────────────┐  ║
              ║   │ 输入股票代码或公司名      │  ║
              ║   │ 例：GOOGL, TSM, "腾讯"   │  ║
              ║   └─────────────────────────┘  ║
              ║                                ║
              ║   选择模型：                    ║
              ║   ◯ DeepSeek (默认)            ║
              ║   ◯ Claude Sonnet              ║
              ║   ◯ 多模型对比                  ║
              ║                                ║
              ║   时间窗口（回测）：            ║
              ║   ◉ 当前  ◯ 历史: ___          ║
              ║                                ║
              ║                  [发起分析]     ║
              ║                                ║
              ╚═══════════════════════════════╝
              
              💡 推荐你研究：
              PLTR · ARM · CRWD（来自最近 BUY 推荐）
```

提交后页面进入 **streaming 全屏沉浸态**：
- 顶部：symbol + 当前 gate (1/7) + 总耗时
- 主区：当前 gate 的流式 thinking 文本 + 工具调用 timeline
- 底部进度条：7 个 gate 的状态点
- 每个 gate 完成后该段 collapse 进顶部，新 gate 接着流
- Gate 7 完成后转为视图 C 报告态

## 5. 编辑式视觉延续

完全沿用 `/dashboard` 的设计 token：
- 字体：Fraunces (display) / Newsreader (body) / JetBrains Mono (data)
- 配色：暖墨 `#15130F` + ink off-white + sage green / terracotta
- 引用 chip 风格保留（已经做了）
- Loading：shimmer + 跳动小圆点
- 空状态：大字斜体衬线 + 引导文案

新增专属语汇：
- **"卷宗 / 档案 / 专题"** 这套档案室词汇代替 "analysis / report / verdict"
- **共识 / 分歧** 一组词代替 "average / consensus"
- **回溯 / 截稿** 时间维度词汇

## 6. 实施建议（如果决定做）

### 必做（核心改造）
1. **路由重构**：拆 4 个 URL（档案柜 / 卷宗 / 单报告 / 发起新分析）
2. **公司卡片聚合**：在档案柜按 symbol 聚合 `analyses[]`
3. **streaming 沉浸态**：把 ThinkingTimeline + ReportPanel 拆成两套独立体验
4. **公司卷宗的 3-slide 轮播**

### 可选（提升体验）
5. **模型分歧矩阵**：跨模型对同一公司的对比矩阵（数据已在 DB 里，需要新 API）
6. **K 线日期标注**：在 K 线上画出每次分析时间点 + verdict
7. **入金/持仓联动**：自动从 SNB 持仓里识别已经买入的股票，标"已持仓"标签
8. **批量发起**：可以一次对 5 支选中的股票批量跑分析

### 不必做（删除）
- 当前的 Layout A/B/C 三态切换 → 直接由路由决定
- 浮动半透明输入框 → 改成专门的 `/new` 入口
- AI 推荐侧栏 tab → 收纳到档案柜的 "+ 发起新分析" 流程里
- 任务调度抽屉 → 暂不做，因为现在还没多少人用

### 工时估算
- **核心改造（1-4）**：~8-12 小时
  - 路由 + 视图 A 聚合卡片（3h）
  - 视图 B 三 slide 轮播（3h）
  - 视图 C breadcrumb + 沉浸态（2h）
  - 视图 D 发起新分析向导 + streaming 沉浸态（4h）
- **可选改造（5-7）**：~6-10 小时
- **总计**：14-22 小时全部

## 7. 不立即重写的理由 / 渐进式推进

如果不想一次大改，**第一步可以只做"档案柜聚合"**：
- 不动 streaming / report 任何一行
- 在 `/company-agent` 的 list mode 下，把右侧的"分析记录列表"改成"按 symbol 聚合的卡片"
- 点击卡片不跳新路由，仍然在右侧打开报告（保留现有逻辑）

这一步 **2-3 小时** 就能让首屏体验有质的飞跃，之后再决定是否继续往下做。

---

## 8. 一句话总结

> **当前的 `/company-agent` 是一个 IDE。重新设计后它会是一个"研究档案室"——你打开它就像打开自己的卷宗柜，每只股票是一份持续累积的研究档案，新的分析是给档案添加最新一稿。**
